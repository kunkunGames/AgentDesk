# PRD Template

문서 버전: `v0`
기준 날짜: `YYYY-MM-DD`
상태: `draft | proposed | accepted | shipped`
소유자: `team/owner`

## 1. 배경

이 문서가 필요한 운영 맥락과 현재 불편을 적는다.

## 2. 문제 정의

- 현재 무엇이 불명확한가
- 어떤 운영 리스크가 있는가
- 어떤 사용자 경험이 깨지고 있는가

## 3. 목표

- 이번 PRD가 고정하려는 계약
- 성공했을 때 달라져야 하는 사용자 경험

## 4. 비목표

- 이번 문서가 의도적으로 다루지 않는 범위

## 5. 사용자 시나리오

- 운영자
- 에이전트 사용자
- 장애 복구/재시작 시나리오

## 6. 옵션

### 옵션 A

- 동작 방식
- 장점
- 단점
- 구현 난이도

### 옵션 B

- 동작 방식
- 장점
- 단점
- 구현 난이도

## 7. 결정

- 채택안
- 채택 이유
- 보류안/폐기안 사유

## 8. UX 계약

- 명령/버튼/상태 메시지
- 세션 전환 규칙
- 장애 시 사용자에게 보이는 동작

## 9. 기술 설계

- 관련 코드 경로
- 저장 상태
- 세션/프로세스/메모리 처리 규칙

## 10. 리스크와 완화

- 기술 리스크
- 운영 리스크
- 롤백 전략

## 11. 구현 단계

1. 설계/상태 계약 반영
2. 런타임 구현
3. 테스트 및 운영 검증

## 12. 검증 기준

- 수동 검증
- 자동 검증
- 로그/메트릭 확인 포인트

## 13. 오픈 질문

- 아직 닫히지 않은 의사결정
