What changed:
Removed a hot-path performance anti-pattern in `src/dispatch/dispatch_create.rs` where the `context_with_session_strategy` JSON was repeatedly serialized to string and parsed back into a map to inject fields like `worktree_target` or `review-decision`. The JSON object is now mutated in-place and serialized exactly once at the end.

Why:
This reduces avoidable allocations and parsing cycles (stringification and string-parsing) during dispatch creation, which is a frequent operation in AgentDesk. It constitutes a deterministic complexity reduction for the dispatch path.

Risk:

If `as_object_mut()` returns `None` (which would only happen if the strategy context somehow wasn't an object, which it already was assumed to be), it safely skips injection rather than panicking, matching previous behavior.
