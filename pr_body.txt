**What changed**
Optimized the cleanup loop for old `long_turn_alert` keys in the `long-turn-monitor.js` timeout policy. The N+1 pattern of fetching keys with `SELECT key FROM kv_meta WHERE key LIKE 'long_turn_alert:%'` and running individual `DELETE` statements on each was replaced with a single deterministic `DELETE FROM kv_meta WHERE key LIKE 'long_turn_alert:%'` query.

**Why**
Repeated database calls in the periodic `onTick1min` timeout path add avoidable IO operations and intermediate memory allocations (holding all matched `key` rows in JS arrays). Replacing it with a single bulk query reduces database round-trips and is a direct O(N) to O(1) complexity reduction.
