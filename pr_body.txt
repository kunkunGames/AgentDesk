**What changed**
Replaced a local `expand_tilde` helper function in `src/services/discord/voice_barge_in.rs` with the existing, centralized `crate::voice::utils::expand_tilde`. The old local function used `dirs::home_dir()` directly and relied on `to_string_lossy()`, while the centralized helper properly works with `Path` objects without lossy conversions or whitespace trimming for valid non-tilde paths.

**Why**
This reduces local complexity and preserves byte-for-byte behavior for non-tilde paths. As highlighted in PR review lessons (e.g. #212), relying on `to_string_lossy` or other string-rebuilding mechanisms for path extension can accidentally trim whitespace or alter paths unexpectedly. Using the correct existing helper ensures byte-for-byte fidelity for valid fallbacks.
