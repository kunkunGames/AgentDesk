## Summary
Added `aria-label` and `aria-pressed` attributes to the color picker buttons in `OfficeManagerModal` and `OfficeManagerView` to improve accessibility for screen readers.

## Accessibility/UX invariant
Ensures icon-only or color-only interactive elements have an accessible name and their state is clearly communicated to assistive technologies.

## User impact
Screen reader users will now know which color they are selecting when managing offices.

## What changed
- Added `aria-label` to the color `<button>` elements in `dashboard/src/components/OfficeManagerModal.tsx`.
- Added `aria-pressed` to reflect selection state in `dashboard/src/components/OfficeManagerModal.tsx`.
- Applied the exact same fixes to the color picker buttons in `dashboard/src/components/OfficeManagerView.tsx`.
- Added `type="button"` to ensure these buttons don't submit forms accidentally.
