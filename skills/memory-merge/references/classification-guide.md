# Phase 3: 분류 가이드

각 memory 파일의 내용을 4계층 중 하나로 분류한다:

| 목적지 | 기준 | 예시 |
|---|---|---|
| **SAM** | 휘발성 + 해당 에이전트만 필요 | 진행 중 작업 상태, 임시 디버그 기록, 세션 컨텍스트 |
| **SAK** | 휘발성 + 전체 에이전트 필요 | 최근 프로젝트 결정, 공유 규칙 변경, 포트/경로 변경 |
| **LTM** | 비휘발성 + 해당 에이전트만 필요 | 전문 절차, 아키텍처 결정 근거, 도메인 패턴 |
| **System Prompt** | 비휘발성 + 전체 에이전트 필요 | 확정된 워크플로우 규칙, 기술 표준, 역할 경계 |

## frontmatter type 기반 힌트
- `type: feedback` → 개인 교정이면 **LTM**, 전체 규칙이면 **SAK** 또는 **System Prompt**(확정적이면)
- `type: project` → 진행 중이면 **SAK**, 확정 결정이면 **LTM** 또는 **System Prompt**
- `type: user` → 대부분 **SAM** 또는 **LTM**
- `type: reference` → 변동 가능하면 **SAK**, 영구적이면 **System Prompt**

## 비휘발성 판단 기준 (수명 테스트)

**핵심 질문: "이 내용의 항목이 완료/소멸/갱신으로 제거되는가?"**

| 수명 | 예시 | 판정 |
|---|---|---|
| **소멸성** — 항목이 완료되면 사라짐 | 이슈 로드맵, 진행 상태, TODO 목록, 스프린트 계획 | 휘발성 → SAM/SAK |
| **갱신성** — 값이 주기적으로 바뀜 | 현재 포트 번호, 활성 브랜치, 임시 환경 설정 | 휘발성 → SAM/SAK |
| **불변성** — 내용이 변하지 않는 결정/패턴 | 아키텍처 결정, 코딩 규칙, 피드백, 도메인 패턴 | 비휘발성 → LTM/System Prompt |

추가 신호:
- 코드/설정에 이미 반영된 확정 사항 → 비휘발성
- "~하기로 했다", "확정" 등의 표현 → 비휘발성
- 날짜 한정, "현재 진행 중", 임시 상태 → 휘발성

**주의:** "프로젝트 결정"이라도 이슈 목록·순서·로드맵처럼 완료 시 항목이 제거되는 내용은 **휘발성(SAM)**이다. 번복 없는 기술 결정만 **비휘발성(LTM)**이다.
