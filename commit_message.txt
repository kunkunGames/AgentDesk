Refiner: reuse expand_user_path in agents API routes

What changed:
Replaced the custom string-based tilde expansion in `src/server/routes/agents_crud.rs` (`resolve_configured_path`) and `src/server/routes/agents_setup.rs` (`expand_tilde`) with the standardized `crate::runtime_layout::expand_user_path` helper.

Why:
To reduce local complexity, avoid duplicating path expansion behaviors across modules, and rely on the centralized layout utilities. The refactor maintains exact byte-for-byte fallback behavior for paths that don't start with `~` or `~/` by validating the prefix explicitly before calling the helper.
