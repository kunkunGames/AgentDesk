Extract analytics route domain from overloaded admin module

- **What changed**: Extracted analytics, stats, and achievements routes from `src/server/routes/domains/admin.rs` into a dedicated `src/server/routes/domains/analytics.rs` module. Updated Axum router merging and inventory documentation.
- **Why**: To enforce a clear domain ownership boundary. The `admin` module was overloaded with observability and reporting routes. Extracting these into an `analytics` module makes route registration and configuration responsibility easier to reason about.

  - Invariant Protected: Public API paths, auth middlewares, and response behavior remain identical (no functional regressions).

  - `python3 scripts/generate_inventory_docs.py`: Successfully regenerated `ARCHITECTURE.md` and `docs/generated/` inventories to match the new files.
