Gatekeeper: make scratch file guard strict across all tracked directories

What changed:
Updated the "Scratch file guard" inside `scripts/ci-script-checks.sh` to use `git ls-files --others --exclude-standard --cached` instead of matching against a static set of paths in the repository root. This allows the guard to successfully detect scratch files (`plan.md`, `test_scratch.rs`, etc.) regardless of how deeply nested they are within directories like `src/`, `docs/`, or `scripts/`. A small drift in generated docs `module-inventory.md` and a cleanup of a stale allowlist entry in `scripts/audit_allowlist.toml` are also included.

Why:
The current scratch file guard relied on basic shell globbing (e.g., `for scratch_file in plan.md...`) which is limited to checking only files situated immediately in the project's root folder. As a result, scratch files created in subdirectories incorrectly skipped validation, defeating the guard's primary purpose.
